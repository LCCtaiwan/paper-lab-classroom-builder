# SPEC：Paper Lab Classroom Builder

## 1. 產品定位

Paper Lab Classroom Builder 是醫療教學課堂產生器。教師提供講義 PDF，系統保留原始講義內容與頁序，自動建立逐頁投影片、建議課程段落與提問停靠點，經教師確認後產生完整課堂網站。

核心價值不是「AI 自動授課」，而是把重複的課堂網站製作工作自動化，同時保留教師對醫療教學內容的最終決定權。

## 2. 目標

教師應能：

1. 只提供一份講義 PDF，即可取得可預覽的課堂草稿。
2. 查看並修改 AI 建議的課程名稱、內容段落與頁段，並自行設定問題整理停靠頁。
3. 選擇是否加入目標 paper、Appendix 與課前 AI 協作導讀。
4. 確認後產生講師後台、純投影頁、學生提問端、六碼加入代碼與 QR code。
5. 保留原始 PDF，不因轉檔或 AI 分析而修改內容。

## 3. 非目標

- 不讓 AI 未經教師確認就發布醫療教學內容。
- 不自動改寫、刪減或重排原講義頁面。
- Pilot 不提供正式帳號、公開自助、計費或大量課程營運能力。
- 不把 Evidence Market 納入預設課堂骨架。
- 不以自動化測試取代教師對段落、問題品質與教學節奏的判斷。

## 4. 使用者與角色

### 教師

- 匯入講義與選填資料。
- 審查 AI 建議。
- 建立並操作課堂。
- 查看、投影或隱藏學生問題。

### 學生

- 掃描 QR code 或輸入六碼加入。
- 查看目前講義頁碼與段落。
- 提問並對相同問題按「同問」。
- 不需建立帳號。

## 5. 輸入

### 必填

- 一份可讀取、未加密的講義 PDF。

### 選填

- 課程名稱。
- 目標 paper PDF。
- Appendix PDF。
- 課前導讀說明或教師自訂 prompt。
- 教師指定的段落與停靠頁。
- 品牌名稱、色彩與課程識別。

## 6. 輸出

- 原始講義 PDF 的未修改副本。
- 依原頁序產生的逐頁投影片影像。
- 課程設定檔。
- 教師確認前的課程預覽。
- Macbook and windows系統講師後台。
- 外接螢幕純投影頁。
- 學生提問與同問頁。
- 六碼課堂代碼與本機產生的 QR code。
- 選配的單一 HTML 課前導讀，可封裝 paper 與 Appendix。
- 可建置、測試與發布的網站專案。

## 7. 核心流程

### 7.1 匯入

1. 驗證 PDF 是否存在、可讀、未加密且頁數有效。
2. 計算原始 PDF checksum，作為內容未修改的基準。
3. 逐頁轉為適合瀏覽器投影的影像，不裁切內容。
4. 抽取每頁文字；掃描型 PDF 才使用 OCR。

### 7.2 AI 建議

AI 根據頁面文字與頁碼提出：

- 課程名稱候選。
- 可能的段落名稱與起訖頁。
- 每段一句內容摘要與可查核的依據頁碼。
- 是否需要 paper、Appendix 或課前導讀的建議。

AI 輸出必須標示為「建議」，不得直接成為已發布內容。

### 7.3 教師確認

教師可：

- 修改課程名稱。
- 合併、拆分、重新命名段落。
- 調整段落起訖頁。
- 由教師新增、刪除或移動問題整理停靠頁。
- 選擇是否加入 paper、Appendix 與課前導讀。

只有教師按下「建立課堂」後，設定才可進入網站產生流程。

### 7.4 建立與發布

1. 依確認後設定產生完整網站。
2. 執行頁面、設定、build、lint 與產品流程測試。
3. 產生課堂預覽。
4. 第一階段輸出本機可執行專案；第二階段才提供線上發布。

## 8. 課程設定檔

第一階段以單一設定檔取代目前寫死的 64 頁、八個段落與固定 paper。

建議結構：

```json
{
  "courseId": "example-course",
  "title": "課程名稱",
  "handout": "content/handout.pdf",
  "paper": "content/paper.pdf",
  "appendix": "content/appendix.pdf",
  "preclassEnabled": true,
  "sections": [
    {
      "id": "section-1",
      "title": "段落名稱",
      "startPage": 1,
      "endPage": 8,
      "summary": "內容摘要",
      "evidencePages": [1, 8]
    }
  ],
  "checkpoints": [8]
}
```

規則：

- `handout` 必填；其他來源選填。
- 頁碼從 1 開始，且不得超出 PDF 頁數。
- 段落不得互相重疊，合計應覆蓋所有講義頁面。
- 停靠頁完全由教師設定，不強制最低數量。
- 設定檔是網站唯一課程內容來源；runtime 不得再寫死頁數或段落數。

## 9. Pilot：跨 Agent Web Builder

### 9.1 建議目錄

```text
paper-lab-classroom-builder/
├── content/
│   ├── handout.pdf
│   ├── paper.pdf          # optional
│   └── appendix.pdf       # optional
├── course.config.json
├── generated/
│   └── handout-pages/
├── site/
│   ├── app/
│   ├── scripts/
│   └── tests/
├── CLAUDE.md
└── docs/
```

### 9.2 建立程序

```text
Web 上傳或放入 PDF
→ 共用匯入與驗證
→ 產生頁面影像與 AI 分段草稿
→ 教師編輯分段並設定停靠頁
→ 確認建立課程
→ 每次授課建立獨立六碼場次
```

### 9.3 第一階段成功條件

- Web、Codex 與 Claude Code 使用同一 Manifest 與 runtime；不修改 runtime 即可建立第二門課。
- 以兩份頁數與結構不同的 PDF 驗證，不得依賴 64 頁或八段假設。
- 兩門課均能完成講師翻頁、外接同步、學生提問、同問、指定問題投影與 QR 加入。

## 10. 正式化階段

Pilot 通過後才開始。新增能力：

- 正式帳號、組織與課程權限。
- 背景轉檔、OCR queue 與重試。
- 多課程搜尋、版本管理與稽核紀錄。
- 公開自助、配額、成本控制與濫用防護。
- 正式保存／刪除政策與一鍵封存。
- 上傳限制、病毒掃描、存取權限與成本控制。

第二階段不得改變「教師確認後才發布」原則。

## 11. 沿用的既有框架

- MacBook 講師後台與外接螢幕純投影模式。
- 同一瀏覽器的即時頁碼同步與伺服器輪詢備援。
- 講義相鄰頁預載。
- 學生提問、同問、問題排序、指定單題投影與隱藏。
- 六碼課堂與本機 QR code。
- MacBook 緊湊後台版面。
- 單一 HTML 課前導讀與選配 PDF 封裝方式。

## 12. 醫療教學與安全原則

- AI 建議必須與原頁碼連結，教師可回看依據。
- AI 產生內容需區分原文事實、教學建議與推論。
- 教師未確認前不得發布。
- 學生端持續提醒不得輸入病人姓名、病歷號或其他個資。
- 上傳檔案不得用於未揭露的模型訓練用途。
- 原始 PDF、轉檔影像與產生內容須有明確保存與刪除政策。

## 13. 非功能需求

- 支援桌面版最新版 Chrome；學生端支援常見手機瀏覽器。
- PDF 轉頁後須維持原比例、頁序與可讀性。
- 同一台電腦的講師端到投影端翻頁目標在 250 ms 內。
- 網路中斷時可使用原始 PDF 作為投影備援。
- 產生流程應可重跑，且相同輸入與設定能得到一致輸出。
- 錯誤訊息須指出是哪個檔案、頁面或設定欄位失敗。

## 14. 驗收條件

### PDF 與內容

- 可讀 PDF 能正確辨識總頁數並產生相同數量的投影片。
- 第一頁、代表性中間頁與最後一頁視覺檢查無裁切、旋轉錯誤或文字不可讀。
- 產生前後原始 PDF checksum 相同。
- 加密、損毀或零頁 PDF 會停止並提供可理解的錯誤。

### 泛用設定

- runtime 不含固定 `64` 頁或固定八段的課程邏輯。
- 更換第二份不同頁數 PDF 與設定檔後，不修改 runtime 即可建立課堂。
- AI 建議包含連續分段、摘要及頁碼依據；停靠頁由教師設定。
- 教師可在建立前完整修改建議。
- 未完成教師確認時，「建立課堂」不可執行。

### 課堂網站

- 講師端、投影端與學生端均依設定顯示正確課名、頁數、段落及停靠點。
- MacBook 翻頁可同步外接投影；學生端在輪詢備援範圍內更新。
- 學生可提問與按同問；問題不會自動投影。
- 講師可指定或收回單一問題，並隱藏不適合的問題。
- QR code 可帶入正確課堂代碼，並保留手動輸入備援。
- Production build、lint 與產品流程測試通過。

### 階段門檻

- 第一份既有講義與第二份全新講義均完成上述驗收，第一階段才可判定 `pass`。
- 第一階段為 `pass` 前，不開始第二階段線上上傳與多課程架構。

## 15. 主要風險

- PDF 只有掃描影像，OCR 造成段落判斷錯誤。
- PDF 版面複雜、直橫頁混合或字型嵌入異常。
- AI 過度解讀頁面，提出與課程目標不符的停靠點。
- paper 與 Appendix 授權不允許公開封裝。
- 線上版本的檔案儲存、轉檔時間與 AI 成本失控。
- 多課程共用同一套即時資料時發生資料隔離問題。

## 16. 待決策事項

1. 第一階段採命令列建立程序，或提供本機圖形化精靈。
2. AI 分析使用哪個模型與資料傳送政策。
3. OCR 工具與繁體中文、英文醫學文件支援範圍。
4. 第二階段是否每門課獨立網站，或共用一個多課程平台。
5. paper／Appendix 預設只供下載，或允許封裝進單一 HTML。
6. 課程設定採 JSON、YAML，或由本機審查頁輸出。

## 17. 建議下一個可驗證切片

先完成「PDF 匯入＋設定檔驅動頁數」：

1. 將 PDF 轉成逐頁影像。
2. 自動產生只包含課名、總頁數與一個預設段落的設定檔。
3. 讓既有講師端與投影端不再依賴固定 64 頁。
4. 用目前講義與第二份不同頁數 PDF 驗證。

此切片 `pass` 後，再加入 AI 段落與停靠點建議。
